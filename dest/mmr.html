The MMR Project

MMR stands for Mike's Metropolis Robot. OK, it's a pretty rubbishy title, but it's one I just made up this minute, and will no doubt be subject to change once I find a really nice name for it, so enough of that.

Anyway, if you've been reading the updates page, you'll know that I've more to update in the Metropolis and less time to update it! So I've started the MMR project to develop a system to automatically keep track the general upkeeping and other fiddly bits of Mike's Metropolis- you know, the usual "Last updated" bits in all the pages, and all the icons next to new or updated links. At the moment I have to do all that by hand- aaargh! But hopefully not for too long now...

The developement of MMR will promise one or two interesting spin-offs. In fact, you can already see one of them in the Contents/Index Page, which has been automatically generated by the early version of MRR1. OK it's pretty sparse at the moment, but for a first go, you can sure sure see the potential.. Future developments should include things like a list of the pages sorted by most recent update, an index of all the URLs referenced in a page, a search facility and a random URL generator.

MRR is current being developed in Perl in DOS (Yuk!) although it's getting test-runs on proper UNIX systems as you can see below.

Current state of MMR

  • 25/7/96

    • MMR 1.5- the update! You can see the results in the latest contents/index page- It's very nice! Both parts (See below) now have a common config file which allows the names of the various files to be configured, and includes an "exclusion list".

      dirscan can now use an exclusion list and also excludes any files not world readable, whilst makecont can now use a template input file to make up it's contents/index page, which include embedded commands for dates, no of pages, alphabetical search and so on.

    • Improvemements to make: makecont will undergo more changes to do stuff like making up an index of pages sorted by most recent update, and a proper hierarcical contents page! I've still to get geturl working.

  • 14/6/96

    • MMR1 is born! This mini-engine is composed of two parts: (Mainly because DOS imposes memory restrictions which prevent Perl from running big programs!)
      • dirscan- this scans the directory tree for HTML and text files (DOS is not very good at this, and keeps including GIFs in it's scan!), and if it's an HTML doc, it'll get it's title. A data file of the filenames+pathnames, the title and the date of change of the HTML docs is saved out.
      • makecont- uses the data file created by dirscan to make the Contents/Index page.
    • Improvemements to make:
      • dirscan- interesting bug to do with symbolic links in UNIX apparently plugged. Other sloppy code bit fixed in the UNIX version. Also added an "exclusion list" of pages to miss out from the scan. Next improvement- only scan files which are readable by everybody.
      • makecont- Basic alphabetical sort added (ASCII really- needs a bit of improvement!). Next- proper hierarcial contents and list of pages sorted by most recent update! Also, more control over the appearance of the HTML page generated.
    • New module! This one's called geturl. It takes the data generated by dirscan and pulls URL tags out of each HTML file. It's not quite at the working stage yet, but when it is, you might see an index of all URLs pointed to by Mike's Metropolis, and a random URL generator!
